03 of 11 · The Problem
Their words, their numbers

Each new carrier API takes one senior engineer a month, and Cargolane has 17 in the queue.

"The integration itself is two weeks of real work. The other two are spent in Postman, in spreadsheets, copying request bodies between teams who never see each other's tests."
Marcus Sundberg, Staff Engineer, Carrier Integrations · March 18 working session

The routing platform team is shipping fewer carrier integrations than committed because every new carrier requires a senior engineer to coordinate across product, ops, and finance, then translate each team's expected behavior into a working API client by hand. Postman collections drift. Spreadsheets get out of sync. The expected payload one team agreed to on Monday is broken by another team's change on Wednesday.

The cost is not the engineering hours alone. It is the deals stuck in pilot because the carrier API is not live yet, plus the routing data Cargolane cannot capture until each integration is done.

05 of 11 · Our Solution
In plain English

FormulaLab is the API client your routing engineers can share with the product and ops teams.

"We turn each carrier API integration into one spreadsheet that product, ops, and engineering all edit together, so you ship in 9 days instead of 30."
1
Spreadsheet-native API client. Each row is a request. Each column is a header, body field, or assertion. Anyone on the team can read it.
2
Built for routing teams. Carrier API patterns, rate-limit handling, and webhook replays ship in the box.
3
What we do not do. We do not host your production traffic. The API client generates the integration code that runs in your stack.
4
What is different. Postman is a developer tool. FormulaLab is a team tool. Product and ops can change a request without filing a ticket.
5
Time to live. Median first-integration ships in 11 days from signed contract. 89% of users send their first real request inside 7 days.

06 of 11 · How It Works
Four steps, all in your existing tools

From a fresh carrier API spec to a shipped integration in 9 working days.

1

Import the spec.

Drop a carrier's OpenAPI file, Postman collection, or PDF. FormulaLab parses it into rows, one row per endpoint, with auth and rate limits already wired.

Outcome · spec loaded in 4 minutes
2

Edit as a team.

Product fills in the business rules. Ops adds the test cases. Engineering writes the assertions. Everyone works in the same view, with row-level comments.

Outcome · spec signed off, 2 days
3

Generate the client.

One click emits a typed Go or TypeScript client that drops into your routing service. Tests come with it, generated from the same rows.

Outcome · PR opened, 1 day
4

Ship and monitor.

Webhook replay catches drift. Schema changes from the carrier fire alerts. Your team edits the row, regenerates, and ships again.

Outcome · live in 6 days
