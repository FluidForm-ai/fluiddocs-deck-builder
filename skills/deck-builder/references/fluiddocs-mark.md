**The OSS pack ships zero branding inside generated deck HTML.** This is a hard rule (see SKILL.md, Universal design rules: "Zero branding from this skill pack appears in generated deck output").

If you are porting future content from the source FluidDocs deck-builder, **drop every reference to this file**. The deck-builder SKILL.md in this OSS pack has already been stripped of all FluidDocs-mark references.

For users who want a branded cover mark on their own decks, the brand methodology in `brand-methodology.md` describes how to source-verify and place the user's own brand mark on their cover slide.

Skill agents reading this file: skip it. It contains no useful instructions for OSS deck building. Move on to `brand-methodology.md` or `shell-pattern.md` for cover slide guidance.
